# Progressive Tool Disclosure

Progressive tool disclosure solves the MCP tool schema bloat problem — where registering 30+ MCP tools burns 60K+ tokens on tool definitions before a single user message is sent.

## The Problem

Every MCP server dumps its full tool definitions into every request. With a typical enterprise setup:

| Scenario | Tool Count | Token Cost |
|----------|-----------|-----------|
| Light user (1-2 MCP servers) | 3-5 | 800-11,000 |
| Power user (3-5 MCP servers) | 10-20 | 2,600-44,000 |
| Enterprise (10+ MCP servers) | 20-50 | 5,200-110,000 |

These schemas are sent on **every request** in a conversation. In a 20-turn session with 40 tools, that's 436K tokens spent purely on tool definitions.

## The Solution

Copium's progressive disclosure loads tool schemas **on-demand** rather than eagerly:

1. **Eager tools** — High-usage tools (file ops, search) are loaded directly
2. **Deferred tools** — Everything else is available via `copium_find_tool`
3. **Cached schemas** — Once discovered, schemas are cached for the session

```python
from copium.mcp_proxy.progressive_disclosure import (
    ProgressiveDisclosureConfig,
    ProgressiveDisclosureInterceptor,
)

config = ProgressiveDisclosureConfig(
    enabled=True,
    eager_load_max=10,
    search_backend="bm25",
    min_tools_for_disclosure=8,
)

interceptor = ProgressiveDisclosureInterceptor(config)

# Intercept outgoing request
modified = interceptor.intercept_request(request_body)
# 61 tools → 10 eager + 2 meta-tools = 12 tools sent

# Handle meta-tool calls from LLM response
response = interceptor.intercept_tool_call("copium_find_tool", {"query": "create github issue"})
```

## Architecture

```
Client (any MCP client)
  │  sends: tools=[61 tools]
  ▼
Copium Proxy
  ├── Schema Registry (indexes all tools)
  ├── Eager Policy (classifies eager vs deferred)
  ├── BM25 Search (tool discovery)
  └── Meta-Tools (find_tool + call_tool)
  │  sends: tools=[10 eager + find_tool + call_tool]
  ▼
LLM Provider (Anthropic/OpenAI/etc.)
```

## Configuration

```python
from copium.mcp_proxy.progressive_disclosure import ProgressiveDisclosureConfig

config = ProgressiveDisclosureConfig(
    # Core settings
    enabled=True,                    # Enable progressive disclosure
    eager_load_max=10,               # Max tools loaded eagerly
    min_tools_for_disclosure=8,      # Threshold to activate

    # Search settings
    search_backend="bm25",           # "bm25" or "keyword"
    search_top_k=3,                  # Results per find_tool query

    # Eager loading patterns
    eager_patterns=[                 # Tool names matching these are always eager
        "read_file", "write_file", "list_dir",
        "search", "grep", "find",
        "run_command", "execute", "bash",
    ],
    eager_prefixes=["copium_"],      # Prefixes always loaded eagerly

    # Behavior
    cache_schemas=True,              # Cache resolved schemas
    inject_system_prompt=True,       # Teach LLM the pattern
)
```

## How the LLM Uses It

The system prompt teaches the LLM a two-step pattern:

1. **Discover:** Call `copium_find_tool(query="create a GitHub issue")`
2. **Execute:** Call `copium_call_tool(tool_name="github_create_issue", arguments={...})`

The LLM adapts to this pattern naturally — it's similar to how Claude Code's `ENABLE_TOOL_SEARCH` works, but transparent and client-agnostic.

## Eager Loading Heuristics

Tools are classified as eager based on:

| Criterion | Priority |
|-----------|----------|
| Matches `eager_prefixes` (e.g., `copium_`) | Highest |
| Previously called in this session | High |
| Matches `eager_patterns` (e.g., `read_file`) | Medium |
| Everything else | Deferred |

If more tools qualify as eager than `eager_load_max`, they're ranked by priority score.

## BM25 Search

The built-in BM25 index provides fast, dependency-free tool discovery:

- Indexes tool names, descriptions, and parameter names
- Handles camelCase and snake_case tokenization
- IDF-weighted scoring with length normalization
- Sub-5ms search latency for typical tool sets
