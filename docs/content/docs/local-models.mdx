# Local Model Integration

Copium provides first-class support for local LLM backends. If you're running models locally, context compression isn't just about saving money — it's about making your agent **functional**. A coding agent that can only fit 5 tool calls before hitting context limits is useless.

## Why Local Models Need Compression MORE

| Factor | Cloud Model | Local Model (8B, 24GB VRAM) |
|--------|-------------|----------------------------|
| Available context | 128K-1M tokens | 8K-32K tokens |
| System prompt overhead | 3-8K (negligible) | 3-8K (25-100% of context!) |
| Tool output per call | 500-5K tokens | Same (500-5K tokens) |
| Max tool calls before full | 50-200 | 2-10 |

**Key insight**: For local models, compression isn't about saving money — it's about making the model functional.

## Quick Start

```bash
# Auto-detect your local backend and configure
copium doctor

# One-command setup for Ollama
copium wrap ollama

# For llama.cpp server
copium wrap llamacpp

# For LM Studio
copium wrap lmstudio
```

## Supported Backends

### Ollama

Copium auto-detects Ollama on `localhost:11434` and queries model capabilities:

```python
from copium.integrations.local import OllamaIntegration

ollama = OllamaIntegration()
if ollama.detect():
    models = ollama.list_models()
    config = ollama.get_proxy_config(model="qwen3:8b")
    print(f"Context: {config['context_window']} tokens")
    print(f"Aggressiveness: {config['compression_aggressiveness']}")
```

### llama.cpp

Connects to llama.cpp's HTTP server and monitors context utilization:

```python
from copium.integrations.local import LlamaCppIntegration

llamacpp = LlamaCppIntegration(server_url="http://localhost:8080")
if llamacpp.detect():
    info = llamacpp.get_server_info()
    print(f"Context: {info.context_size} tokens")
    print(f"Used: {info.context_used} tokens")
    print(f"Utilization: {llamacpp.get_context_utilization():.1%}")
```

### LM Studio

Auto-detects LM Studio's OpenAI-compatible API:

```python
from copium.integrations.local import LMStudioIntegration

lmstudio = LMStudioIntegration()
if lmstudio.detect():
    model = lmstudio.get_active_model()
    config = lmstudio.get_proxy_config(model=model.id)
```

### Auto-Detection

Scan for all running backends at once:

```python
from copium.integrations.local import detect_local_backends

backends = detect_local_backends()
for backend in backends:
    print(f"{backend.backend_type}: {backend.url} ({backend.model})")
    config = backend.get_proxy_config()
```

## The Smart Zone

Research shows LLMs degrade sharply after 40% context utilization:

- **0-40%**: "Smart Zone" — model performs at peak
- **40-70%**: Degradation begins — model starts losing track
- **70-100%**: "Dumb Zone" — model hallucinates, forgets instructions

For a 32K context local model, the Smart Zone is only **12.8K tokens**. That's system prompt (3-5K) + a few tool calls. Compression is the only way to keep the agent in the Smart Zone.

## VRAM-Aware Compression

### Adaptive Compression

The `AdaptiveCompressor` monitors GPU memory and adjusts in real-time:

```python
from copium.integrations.local import AdaptiveCompressor

compressor = AdaptiveCompressor()

# Based on current VRAM pressure
config = compressor.get_config()
print(f"Level: {config.level}")  # light/standard/aggressive/maximum
print(f"Reason: {config.reason}")

# Based on context constraints
config = compressor.get_config_for_context(
    context_size=32768,
    context_used=20000,
    kv_cache_precision="q4_0",
)
```

### Hardware Presets

Pre-configured for common GPU setups:

```python
from copium.integrations.local import get_preset_for_vram, PRESETS

# Auto-select based on detected VRAM
preset = get_preset_for_vram(vram_mb=8192)
print(f"Preset: {preset.name}")
print(f"Transforms: {preset.transforms}")

# Or use a named preset directly
preset_8gb = PRESETS["8gb"]
config = preset_8gb.to_proxy_config()
```

| GPU VRAM | Preset | Transforms | Smart Zone |
|----------|--------|------------|------------|
| ≤8GB | `8gb` | All 6 transforms | 35% |
| ≤12GB | `12gb` | 5 transforms | 40% |
| ≤16GB | `16gb` | 4 transforms | 40% |
| ≤24GB+ | `24gb` | 2 transforms | 45% |

## KV Cache Awareness

### The Precision Cliff

KV cache quantization (Q4/Q8) causes accuracy collapse at high context:

| KV Cache Type | Without Copium | With Copium | Improvement |
|---------------|----------------|-------------|-------------|
| FP16 | 32K usable | 32K usable | 0% (already optimal) |
| Q8_0 | 30K before issues | 48K stable | +60% |
| Q4_0 | 20K before issues | 35K stable | +75% |

### Detection

Copium detects KV cache precision from:
1. Explicit config (`kv_cache_precision` setting)
2. Environment variables (`OLLAMA_KV_CACHE_TYPE`, `VLLM_KV_CACHE_DTYPE`)
3. Backend API endpoints (`/props`, `/health`)

## Smart Routing (Triage Engine)

Route simple tasks locally, compress complex tasks for cloud:

```python
from copium.integrations.local import LocalTriageEngine

engine = LocalTriageEngine(
    local_model="qwen3:8b",
    cloud_model="claude-sonnet-4-20250514",
    triage_threshold=0.7,
)

decision = await engine.route(messages)
if decision.is_local:
    # Handle locally — fast, free
    print(f"Local: {decision.reason}")
else:
    # Compress and send to cloud
    print(f"Cloud: {decision.reason} (complexity: {decision.complexity_score:.2f})")
```

### How Triage Works

1. **Heuristic scoring** (instant): message length, keywords, code blocks
2. **LLM judgment** (for ambiguous cases): local model rates complexity 0-1
3. **Blended score**: 40% heuristic + 60% LLM judgment
4. **Decision**: below threshold → local, above → cloud with compression

### Expected Savings

| Workload | Cloud Tokens Saved | Quality Impact |
|----------|-------------------|----------------|
| Edit-heavy (grep, find, replace) | 45-79% | None |
| Explanation-heavy | 40-60% | None |
| RAG-heavy | 20-40% | Minor (triage latency) |
| General chat | 30-50% | None |

## Streaming Compression

For environments where every megabyte of RAM matters:

```python
from copium.integrations.local import StreamingCompressor

compressor = StreamingCompressor(chunk_size=4096)

# Process chunk-by-chunk (never buffers full content)
for chunk in compressor.compress_iter(large_context):
    if not chunk.is_critical:
        print(f"Compressed: {chunk.original_size} → {chunk.compressed_size}")

# Check overall stats
print(f"Savings: {compressor.stats.savings_pct:.1f}%")
```


```bash
# Tell Copium about your KV cache type
export OLLAMA_KV_CACHE_TYPE=q4_0
export LLAMA_CPP_KV_CACHE_TYPE=q8_0
export VLLM_KV_CACHE_DTYPE=fp8

# Custom backend URLs

export COPIUM_LLAMACPP_URL=http://localhost:8080
```
